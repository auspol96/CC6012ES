## ðŸ§  4. Key Rules for DFD Design

1. **Each process must have both input and output**
   â†’ Data cannot disappear or appear from nowhere.
2. **External entities** are always outside the system boundary.
   â†’ They provide or receive data, but do not process it.
3. **Data stores** represent where data is kept permanently or temporarily.
   â†’ They connect only to processes (not directly to external entities).
4. **Processes** are shown as **circles** or **rounded rectangles**.
   â†’ Each process should represent one clear function.
5. **Data flows** use arrows to show direction of information movement.
   â†’ The arrow label should name the data being transferred.
